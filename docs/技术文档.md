## 1. 项目概述与技术栈

**目标**：将现有的 React + Vite 前端项目，无缝集成到 Windows 原生桌面应用中，享有本地系统调用能力（遥测、文件处理等），并可正常与远程 Go 后端服务通信，最终通过 MSIX 打包分发。

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | React 18+, Vite | SPA 应用，静态文件加载 |
| 桌面壳 | WinUI 3 (Windows App SDK) | 原生窗口框架 |
| 浏览器内核 | WebView2 (Edge Chromium) | 嵌入 Web 内容 |
| 本地逻辑 | C# (.NET 6/8) | 桥接调用系统 API |
| 远程后端 | Go (Gin / Echo 等) | RESTful / WebSocket |
| 打包方式 | MSIX 打包项目 | 现代化包管理，自动更新，干净卸载 |
| 工程化 | MSBuild、npm scripts | 自动复制前端产物、多环境切换 |

## 4. WinUI 3 宿主项目创建与配置

### 4.1 创建项目
1. 打开 Visual Studio 2022 → 新建项目 → 搜索 “WinUI” → 选择 **“空白应用，打包(桌面中的 WinUI 3)”**。
2. 命名为 `MyDesktopApp.Host`，放在 `src/` 目录，解决方案命名为 `MyDesktopApp`。

### 4.2 安装 WebView2 NuGet 包
- 右键 `MyDesktopApp.Host` 项目的“引用” → 管理 NuGet 包 → 搜索 **Microsoft.Web.WebView2** → 安装最新稳定版（如 `1.0.2903.40`）。

### 4.3 修改 MainWindow.xaml
将默认的 `StackPanel` 替换为全窗口 WebView2 控件：
```xml
<Window x:Class="MyDesktopApp.Host.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        xmlns:controls="using:Microsoft.UI.Xaml.Controls">
    <Grid>
        <controls:WebView2 x:Name="MainWebView" 
                           Source="about:blank"
                           HorizontalAlignment="Stretch" 
                           VerticalAlignment="Stretch"/>
    </Grid>
</Window>
```

### 4.4 初始化 WebView2 并加载前端
修改 `MainWindow.xaml.cs`，添加窗口加载事件处理：
```csharp
using Microsoft.UI.Xaml;
using Microsoft.Web.WebView2.Core;

namespace MyDesktopApp.Host;

public sealed partial class MainWindow : Window
{
    public MainWindow()
    {
        this.InitializeComponent();
        Loaded += MainWindow_Loaded;
    }

    private async void MainWindow_Loaded(object sender, RoutedEventArgs e)
    {
        await InitializeWebViewAsync();
    }

    private async System.Threading.Tasks.Task InitializeWebViewAsync()
    {
        await MainWebView.EnsureCoreWebView2Async();

        #if DEBUG
        // 开发模式：加载 Vite 开发服务器（支持热更新）
        MainWebView.CoreWebView2.Navigate("http://localhost:5173");
        // 打开 DevTools 方便调试
        MainWebView.CoreWebView2.OpenDevToolsWindow();
        #else
        // 生产模式：使用虚拟主机映射加载本地文件
        MainWebView.CoreWebView2.SetVirtualHostNameToFolderMapping(
            "app.local",           // 虚拟域名
            "Assets/Web",          // 前端静态文件所在文件夹
            CoreWebView2HostResourceAccessKind.Allow);
        MainWebView.CoreWebView2.Navigate("https://app.local/index.html");
        #endif
    }
}
```

### 4.5 配置 .csproj 自动化复制前端产物
打开 `MyDesktopApp.Host.csproj`，在 `<Project>` 节点内添加：
```xml
<ItemGroup>
    <!-- 确保 Assets/Web 下的所有文件作为内容并复制到输出目录 -->
    <Content Include="Assets\Web\**\*" CopyToOutputDirectory="PreserveNewest" />
</ItemGroup>

<!-- 构建前自动将 React 的 dist 目录复制到 Assets/Web -->
<Target Name="CopyFrontendDist" BeforeTargets="BeforeBuild">
    <ItemGroup>
        <FrontendFiles Include="..\MyDesktopApp.Frontend\dist\**\*" />
    </ItemGroup>
    <Copy SourceFiles="@(FrontendFiles)" 
          DestinationFolder="Assets\Web\%(RecursiveDir)" 
          SkipUnchangedFiles="true" 
          Condition="Exists('..\MyDesktopApp.Frontend\dist')"/>
</Target>
```
这样每次编译宿主项目时，最新的前端构建产物都会同步到输出目录。

`.env.production`:

## 6. WebView2 双向通信桥接层设计

### 6.1 整体设计
- **前端 → C#**：使用 `window.chrome.webview.postMessage` 发送 JSON。
- **C# → 前端**：通过 `ExecuteScriptAsync` 调用 JS 全局函数。
- **高级**：使用 `AddHostObjectToScript` 暴露 C# 对象给 JS，实现同步/异步调用。

### 6.2 前端封装 (`src/bridge/nativeBridge.ts`)
```typescript
const isInWebView = typeof window.chrome?.webview !== 'undefined';

export const nativeBridge = {
  postMessage: (type: string, payload?: any) => {
    const message = JSON.stringify({ type, payload });
    if (isInWebView) {
      window.chrome.webview.postMessage(message);
    } else {
      console.log('[Mock Native]', type, payload);
    }
  },

  onMessage: (callback: (data: any) => void) => {
    if (isInWebView) {
      window.chrome.webview.addEventListener('message', (event: any) => {
        try {
          callback(JSON.parse(event.data));
        } catch {
          callback(event.data);
        }
      });
    }
  },

  invokeHostAsync: async (methodName: string, ...args: any[]) => {
    if (isInWebView) {
      const hostObject = window.chrome.webview.hostObjects.native;
      return await hostObject[methodName](...args);
    }
    throw new Error('Not in WebView2 environment');
  }
};
```

### 6.3 C# 端接收消息
在 `MainWindow.xaml.cs` 的 `InitializeWebViewAsync` 中添加：
```csharp
MainWebView.CoreWebView2.WebMessageReceived += (sender, args) =>
{
    var msg = args.TryGetWebMessageAsString();
    var handler = new WebMessageHandler(MainWebView);
    handler.Handle(msg);
};
```

创建 `Bridge/WebMessageHandler.cs`：
```csharp
using Microsoft.Web.WebView2.Core;

namespace MyDesktopApp.Host.Bridge;

public class WebMessageHandler
{
    private readonly CoreWebView2 _webView;
    public WebMessageHandler(WebView2 webView) => _webView = webView.CoreWebView2;

    public void Handle(string rawMessage)
    {
        // 解析 JSON，调用对应的服务，然后通过 ExecuteScriptAsync 返回结果
        // 示例：{"type":"telemetry","payload":{}}
        // ...
    }
}
```

### 6.4 暴露 HostObject（可选，更直接）
`Bridge/NativeHostObject.cs`:
```csharp
using System.Runtime.InteropServices;

namespace MyDesktopApp.Host.Bridge;

[ComVisible(true)]
public class NativeHostObject
{
    public string GetMachineName() => Environment.MachineName;
    public string GetSystemVersion() => Environment.OSVersion.ToString();
}
```
在 WebView2 初始化后注册：
```csharp
MainWebView.CoreWebView2.AddHostObjectToScript("native", new NativeHostObject());
```
前端直接调用：
```typescript
const machine = await window.chrome.webview.hostObjects.native.GetMachineName();
```

## 7. 本地逻辑实现（遥测、数据处理等）

### 7.1 定义服务接口与模型
`Models/TelemetryRecord.cs`:
```csharp
namespace MyDesktopApp.Host.Models;

public class TelemetryRecord
{
    public double CpuUsage { get; set; }
    public double MemoryUsageMB { get; set; }
    public DateTime Timestamp { get; set; }
}
```

`Services/ITelemetryService.cs`:
```csharp
namespace MyDesktopApp.Host.Services;

public interface ITelemetryService
{
    TelemetryRecord Collect();
}
```

`Services/TelemetryService.cs`:
```csharp
using System.Diagnostics;
using MyDesktopApp.Host.Models;

namespace MyDesktopApp.Host.Services;

public class TelemetryService : ITelemetryService
{
    private PerformanceCounter _cpuCounter = new("Processor", "% Processor Time", "_Total");

    public TelemetryRecord Collect()
    {
        var process = Process.GetCurrentProcess();
        return new TelemetryRecord
        {
            CpuUsage = Math.Round(_cpuCounter.NextValue(), 2),
            MemoryUsageMB = Math.Round(process.WorkingSet64 / 1024.0 / 1024.0, 2),
            Timestamp = DateTime.Now
        };
    }
}
```

### 7.2 依赖注入配置
在 `App.xaml.cs` 中搭建 DI 容器：
```csharp
using Microsoft.Extensions.DependencyInjection;
using MyDesktopApp.Host.Services;

public partial class App : Application
{
    public static IServiceProvider Services { get; private set; }

    protected override void OnLaunched(LaunchActivatedEventArgs args)
    {
        var services = new ServiceCollection();
        services.AddSingleton<ITelemetryService, TelemetryService>();
        // 注册其他服务...
        Services = services.BuildServiceProvider();

        // 启动窗口
        _ = new MainWindow();
        base.OnLaunched(args);
    }
}
```

在窗口或桥接类中通过 `App.Services.GetRequiredService<ITelemetryService>()` 获取实例。

## 9. 与 Go 后端 API 交互

WebView2 中的前端与标准浏览器完全一致，使用 `fetch` 或 Axios 调用 Go 后端 API。

### 9.1 React 示例
```typescript
const API_BASE = import.meta.env.VITE_API_BASE_URL;

export async function fetchData() {
  const res = await fetch(`${API_BASE}/api/v1/data`);
  if (!res.ok) throw new Error('Network error');
  return res.json();
}
```

### 9.2 Go 后端 CORS 配置（Gin 示例）
必须允许来自 `https://app.local`（生产）和 `http://localhost:5173`（开发）的请求：
```go
func CORSMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        allowed := map[string]bool{
            "http://localhost:5173": true,
            "https://app.local":     true,
        }
        origin := c.Request.Header.Get("Origin")
        if allowed[origin] {
            c.Header("Access-Control-Allow-Origin", origin)
            c.Header("Access-Control-Allow-Credentials", "true")
            c.Header("Access-Control-Allow-Headers", "Content-Type, Authorization")
        }
        if c.Request.Method == "OPTIONS" {
            c.AbortWithStatus(204)
            return
        }
        c.Next()
    }
}
```

### 9.3 自签名证书
- 开发阶段可在 WebView2 创建时使用 `--ignore-certificate-errors`（仅调试）。
- 生产环境务必使用可信 CA 签发的证书。

## 10. 打包与部署（MSIX 打包）

MSIX 是现代 Windows 应用包，具有干净安装/卸载、自动更新、依赖管理等特点。

### 10.1 创建 MSIX 打包项目
1. 在解决方案中右键 → 添加 → 新建项目 → 搜索 **“打包项目”**（Windows Application Packaging Project）。
2. 命名为 `MyDesktopApp.Package`，位置放在 `src/` 目录。
3. 打包项目创建后，右键其“应用程序”节点 → 添加引用 → 勾选 `MyDesktopApp.Host`。

### 10.2 配置包清单 (`Package.appxmanifest`)
在打开的清单编辑器中设置：
- **应用程序名称**：你的应用显示名
- **描述**：简要说明
- **默认语言**：zh-CN（或根据需要）
- **起始页**：会自动由宿主项目决定，无需更改
- **视觉资产**：替换 `Assets` 文件夹中的图标，例如 `StoreLogo.png`（尺寸 50x50、100x100 等）
- **目标设备系列**：Windows.Desktop
- **最低版本**：Windows 10 version 1809 (10.0.17763.0)
- **声明**：在“声明”选项卡中，可添加 `windows.packageDependency` 自动声明对 Windows App SDK 的依赖（通常项目创建时已配置）

### 10.3 确保 WebView2 运行时依赖
推荐在打包项目文件 `Package.wapproj` 中启用常青引导程序，这样用户缺失 WebView2 时会自动下载安装：
```xml
<PropertyGroup>
    <!-- 启用 WebView2 常青引导程序 -->
    <WebView2EnableEvergreenBootstrapper>true</WebView2EnableEvergreenBootstrapper>
</PropertyGroup>
```
如果未自动引入，可添加 NuGet 引用：
```xml
<ItemGroup>
    <PackageReference Include="Microsoft.Web.WebView2" Version="1.0.2903.40" />
</ItemGroup>
```

### 10.4 构建 MSIX 包
1. 在 VS 中将 `MyDesktopApp.Package` 设为启动项目。
2. 确保配置为 **Release**，平台选择 **x64**（或 ARM64）。
3. 确保前端已构建（`npm run build`），宿主项目构建时自动复制到 `Assets/Web`。
4. 右键打包项目 → 发布 → 创建应用包。
5. 分发方式选择 **“旁加载”**（若不上架商店），勾选“启用自动更新”并设置更新路径（如 `https://update.mysite.com/appinstaller/`）。
6. 点击“创建”，选择输出位置，并勾选“使用当前时间戳生成自签名测试证书”（开发用）。
7. 生成后得到：
   - `MyDesktopApp.Package_1.0.0.0_x64.msix`（安装包）
   - `MyDesktopApp.Package_1.0.0.0_x64.appinstaller`（用于自动更新）
   - `index.html` 及 `appinstaller` 可部署到服务器供用户获取。

### 10.5 安装与测试
- 双击 `.msix` 文件安装。
- 如果是自签名证书，需先将证书导入到“受信任的根证书颁发机构”或“受信任人”存储。双击 `.cer` 文件 → 安装证书 → 选择“本地计算机” → “受信任的根证书颁发机构”。
- 安装后应用出现在开始菜单，卸载可通过设置 → 应用 → 找到并卸载。

### 10.6 自动更新分发
将生成的 `.appinstaller` 和 `.msix` 文件上传到服务器，用户安装 `.appinstaller` 后，后续版本更新会自动检测并安装。

## 11. 性能优化与最佳实践

### 11.1 启动速度
- 使用 `EnsureCoreWebView2Async` 后立刻导航，避免阻塞 UI。
- 可添加 Splash Screen（闪屏窗口），在 WebView2 初始化期间展示品牌背景。
- 窗口最小化时调用 `MainWebView.CoreWebView2.TrySuspend()`，恢复时 `TryResume()` 节省 CPU。

### 11.2 内存优化
- 虚拟主机映射比本地 HTTP 服务器节省一个进程开销。
- 高频通信（如每秒几十次）建议批量处理，减少 JS ↔ C# 边界切换。
- 对于不活跃的 WebView2 实例，适时销毁重建。

### 11.3 前端优化
- 使用 React.lazy、动态 import 进行代码分割。
- 图片使用 WebP 格式，配合懒加载。
- 避免主线程大量计算，可借助 Web Worker（WebView2 完全支持）。

### 11.4 本地服务优化
- 遥测/数据处理放在后台线程，通过 `CoreWebView2.Dispatcher` 回到 UI 线程更新。
- 数据库使用 SQLite 的 `Microsoft.Data.Sqlite` 或 LiteDB，轻量且高效。

### 12.3 常见问题排查表

| 问题现象 | 可能原因 | 解决方法 |
|---------|----------|---------|
| 页面空白 / 404 | 虚拟主机路径错误 | 检查 `Assets/Web` 下是否有 `index.html`，映射域名与导航是否一致 |
| CORS 错误 | 后端不允许 WebView 的源 | 在 Go 后端 CORS 中间件中添加 `https://app.local` |
| `postMessage` 无响应 | 未注册 `WebMessageReceived` | 确保 `MainWebView.CoreWebView2.WebMessageReceived += handler` 已执行 |
| 打包后无法运行 | 缺少 WebView2 Runtime | 在 `.wapproj` 中启用 `WebView2EnableEvergreenBootstrapper` |
| 后端请求超时 | 代理或网络限制 | 检查 WebView2 默认使用系统代理，可通过 `CoreWebView2EnvironmentOptions` 指定 `--proxy-server` |
| 自签名证书错误 | WebView2 信任系统证书 | 将自签名证书导入“受信任的根证书颁发机构”，或开发时临时使用 `--ignore-certificate-errors` |
